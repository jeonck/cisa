# 패치 및 변경 관리
**Patch & Change Management**

:::info 관련 표준
CISA Domain 4.3 / ITIL v4 Change Control / NIST SP 800-40
:::

:::note 작성 예정
이 문서는 작성 중입니다.
:::

## 주요 토픽
- 패치 관리 절차 — 식별, 테스트, 배포, 검증
- 가동 중단 최소화를 위한 패치 배포 전략
- 표준 변경 / 일반 변경 / 긴급 변경(Emergency Change) 분류
- 긴급 변경 통제 및 사후 승인(Post-Implementation Review) 프로세스
- 변경 실패율(Change Failure Rate) KPI 관리
